## 3. Firebase Rules

### Findings
- `database.rules.json`: writes require auth for production paths
- `escrow` path: enforce `covenant_id` and `tx_hash` presence
- `players`: limit to 2 per escrow
- Root `.write` allows public auth fallback for dev

### Recommendation (Medium)
Tighten root rules for production:
```json
{
  "rules": {
    ".read": false,
    ".write": false,
    "escrows": { ".read": "auth != null", ".write": "auth != null" }
  }
}
```

**Status:** PASS with recommendation

## 4. Spend Paths (Covenant Enforcement)

### Findings
- All spend paths in `htp-rust-backend/src/wasm/covenant_escrow.rs`:
  - Winner: attestor + winner dual sig -> 98% winner + 2% fee (enforced in script + Rust)
  - Deadline: `OP_CHECKBLOCKHEIGHT` at block > deadline -> split refund
  - Cancel: creator-only sig before join
  - Leave: opponent auto-wins (no action needed)
- Protocol fee address hardcoded in Rust, not mutable from JS

**Status:** PASS

## 5. Fee Integrity (u64 SOMPI)

### Findings
- All fee calculations in `fee_engine.rs` use `u64` SOMPI only
- No `parseFloat`/`toFixed` on KAS amounts in Rust
- Formulas match spec:
  - `skill_win`: pot * 0.98 / 0.02
  - `event_win_std`: stake * odds * 0.98 / 0.02
  - `hedge_claim`: 70/30 split
  - `cancel`: 100% refund
  - `deadline`: split refund

### Verification
```bash
grep -r "parseFloat\|toFixed" htp-rust-backend/src/ | wc -l  # 0 in Rust
grep -r "parseFloat" --include="*.js" . | grep -v node_modules | wc -l  # Only in wallet UI (harmless)
```

**Status:** PASS
